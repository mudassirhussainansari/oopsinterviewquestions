# **1Ô∏è‚É£ What is OOP?**

**Answer:**
OOP (Object-Oriented Programming) is a programming paradigm based on the concepts of **objects** and **classes**. It helps in writing modular, reusable, and maintainable code.
Java implements OOP through principles like **Encapsulation, Inheritance, Polymorphism, and Abstraction**.

---

# **2Ô∏è‚É£ What is a Class and an Object?**

### **Class**

A blueprint/template that defines properties and behaviors.

### **Object**

A real instance created from a class.

**Example:**

```java
class Car {
   String brand;
   void start() {}
}

Car c = new Car(); // object
```

---

# **3Ô∏è‚É£ What is Encapsulation?**

**Answer:**
Encapsulation means **binding data (variables) and methods (functions)** into a single unit and restricting direct access using **private** fields and **public getters/setters**.

**Why used?**
‚úî Protects data
‚úî Controls access
‚úî Improves maintainability

---

# **4Ô∏è‚É£ What is Inheritance?**

**Answer:**
Inheritance allows one class to acquire the properties and methods of another class using the `extends` keyword.

**Example:**

```java
class Animal { void eat() {} }
class Dog extends Animal { void bark() {} }
```

**Benefits:**
‚úî Code reusability
‚úî Method overriding
‚úî Supports hierarchical relationships

---

# **5Ô∏è‚É£ What is Polymorphism?**

### **Answer:**

Polymorphism means **one name, many forms**.

### Two types:

1. **Compile-time polymorphism (Method Overloading)**
2. **Runtime polymorphism (Method Overriding)**

**Example:**

```java
class A { void show() {} }
class B extends A { void show() {} }  // overriding
```

---

# **6Ô∏è‚É£ What is Method Overloading?**

**Answer:**
Method Overloading happens when multiple methods share the same name but differ in **parameters**.

**Occurs at:** Compile-time.

---

# **7Ô∏è‚É£ What is Method Overriding?**

**Answer:**
When a subclass provides its own implementation of a method defined in the parent class, it‚Äôs overriding.

**Occurs at:** Runtime.

---

# **8Ô∏è‚É£ What is Abstraction?**

**Answer:**
Abstraction hides implementation details and shows only necessary features.

**Achieved by:**
‚úî Abstract Class
‚úî Interface

---

# **9Ô∏è‚É£ Abstract Class vs Interface**

| Feature     | Abstract Class      | Interface                                   |
| ----------- | ------------------- | ------------------------------------------- |
| Methods     | Abstract + Concrete | Abstract (Java 8+ default & static allowed) |
| Variables   | Final or non-final  | Always `public static final`                |
| Inheritance | Single              | Multiple                                    |
| Use Case    | Partial abstraction | Full abstraction                            |

# **üîü What is a Constructor?**

**Answer:**

**Types:**

1. Default
2. Parameterized
3. Copy constructor (manual in Java)

# **1Ô∏è‚É£1Ô∏è‚É£ What is ‚Äòsuper‚Äô keyword?**

**Answer:**

# **1Ô∏è‚É£2Ô∏è‚É£ What is ‚Äòthis‚Äô keyword?**

**Answer:**
`this` refers to the current object.

Used for:
‚úî Avoiding variable shadowing

‚úî Calling constructors (`this()`)

# **1Ô∏è‚É£3Ô∏è‚É£ Can Java support multiple inheritance?**

**Answer:**
Java does **not** support multiple inheritance using classes (to avoid ambiguity), but it **supports it through interfaces**.

# **1Ô∏è‚É£4Ô∏è‚É£ What are Access Modifiers?**

* **public** ‚Äì accessible everywhere
* **private** ‚Äì accessible only within the class
* **protected** ‚Äì accessible in same package + subclasses
* **default** ‚Äì package-private

# **1Ô∏è‚É£5Ô∏è‚É£ What is the difference between Overloading and Overriding?**

| Overloading                     | Overriding                 |
| ------------------------------- | -------------------------- |
| Compile-time                    | Runtime                    |
| Same name, different parameters | Same name, same parameters |
| Same class                      | Parent-child classes       |

# **1Ô∏è‚É£6Ô∏è‚É£ What is Composition?**

**Answer:**
Composition is a design principle where one class is made of one or more objects of other classes.

**Example:**

```java
class Engine {}
class Car {
   private Engine engine = new Engine();
}
```

**Strong relationship:** If Car is destroyed, engine also dies.

---

# **1Ô∏è‚É£7Ô∏è‚É£ What is Aggregation?**

**Answer:**
Aggregation is a weak association where one class has a reference to another but can exist independently.

Example:

```java
class Department {}
class Student {
   Department dept;
}
```

---

# **1Ô∏è‚É£8Ô∏è‚É£ What is the difference between Encapsulation and Abstraction?**

| Achieved using private + getters/setters | Achieved using abstract class & interface |
| Focuses on security                      | Focuses on design                         |
